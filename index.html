Тестирование кода

Тестирование

процесс проверки и оценки функциональности, производительности и надежности программного обеспечения с целью выявления ошибок и их устранения

Зачем тестировать код?

  • Обнаружение ошибок на ранних этапах разработки.
  • Уверенность в том, что изменения не нарушили существующий функционал.
  • Повышение качества программного обеспечения.
  • Удовлетворенность пользователей: приложение работает так, как ожидается.

Типы тестирования программного обеспечения

Модульное тестирование

Проверка отдельных функций или компонентов программы

Интеграционное тестирование

Проверка взаимодействия между компонентами системы

Системное тестирование

Проверка всей системы в целом на соответствие требованиям и ожиданиям

Приемочное тестирование

Проверка программного обеспечения с точки зрения конечного пользователя

Модуль unittest

встроенная библиотека для написания и запуска тестов в Python 


import unittest

Тест кейс

набор условий и шагов, которые определяют, как следует проверять конкретную функциональность программы, а также ожидаемый результат

Состоит из:

  • Описание тестируемой функции
  • Входные данные
  • Шаги выполнения
  • Ожидаемый результат
  • Реальный результат

Тестовые случаи

TestCase

который используется для группировки тестов

Каждое тестовое задание внутри этого класса представляет собой проверку какого-то фрагмента кода 

import unittest

class MyTestCase(unittest.TestCase):
    def test_addition(self):
        self.assertEqual(2 + 2, 4)

    def test_subtraction(self):
        self.assertEqual(5 - 3, 2)

Тестовые методы

Методы класса TestCase , начинающиеся с префикса test_ , которые и выполняются как тесты 


import unittest

class MyTestCase(unittest.TestCase):
    def test_addition(self):
        self.assertEqual(2 + 2, 4)

    def test_subtraction(self):
        self.assertEqual(5 - 3, 2)

Методы утверждений (assertions)

Используются для проверки корректности работы кода

  • assertEqual(a, b) — проверяет, равны ли значения a и b.
  • assertTrue(x) — проверяет, что выражение x истинно.
  • assertFalse(x) — проверяет, что выражение x ложно.
  • assertRaises(exception, func) — проверяет, что функция func вызывает исключение exception.

Запуск тестов

Тесты можно запускать с помощью команды unittest.main() , которая автоматически находит все методы с префиксом test_ и запускает их 


python -m unittest test_module.py

Работа с данными

unittest предоставляет два метода, которые могут быть полезны для подготовки тестов и очистки после их выполнения

setUp()

используется для подготовки данных или состояний, необходимых для выполнения теста

tearDown()

используется для очистки ресурсов, например, закрытия соединений с базой данных 


import unittest

class MyTestCase(unittest.TestCase):
    def setUp(self):
        self.test_data = [1, 2, 3]
    
    def tearDown(self):
        self.test_data = None
    
    def test_sum(self):
        self.assertEqual(sum(self.test_data), 6)

Запуск тестов

unittest.main()

метод автоматически найдет все тестовые методы в вашем классе и выполнит их 


if __name__ == '__main__':
    unittest.main()

Пример использования реальной функции


def add_numbers(numbers):
    return sum(numbers)
                    

import unittest
from my_module import add_numbers

class AddNumbersTestCase(unittest.TestCase):
    def test_add_positive_numbers(self):
        self.assertEqual(add_numbers([1, 2, 3]), 6)

    def test_add_empty_list(self):
        self.assertEqual(add_numbers([]), 0)

    def test_add_mixed_numbers(self):
        self.assertEqual(add_numbers([-1, 1, 2]), 2)

    def test_add_large_numbers(self):
        self.assertEqual(add_numbers([1000000, 2000000]), 3000000)

Структура вывода результатов

При запуске тестов с помощью unittest, результаты тестирования выводятся в консоль.

  • . — тест прошел успешно.
  • F — тест не прошел (неудача).
  • E — возникла ошибка (исключение во время выполнения теста)
